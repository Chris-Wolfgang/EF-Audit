Wolfgang.AuditTrail

Namespace Wolfgang.AuditTrail

Classes

AuditOptions

Configures how the audit interceptor captures changes and how the schema installer creates the audit tables.

AuditSaveChangesInterceptor

Auto-transaction ISaveChangesInterceptor that captures Insert / Update / Delete operations via ChangeTracker and writes audit header + detail rows in the same transaction as the user's save.

AuditValueColumn

Describes a column that an IAuditValueSerializer requires on the AuditDetail table.

AuditingDbContext

Base DbContext that writes audit rows atomically with every SaveChanges() / SaveChangesAsync(CancellationToken) call. Derive your application's context from this instead of DbContext and continue calling SaveChangesAsync as usual — audit capture happens transparently in the same transaction.

DbContextOptionsBuilderExtensions

EF Core DbContextOptionsBuilder extensions that wire the AuditSaveChangesInterceptor into a consumer's DbContext.

ModelBuilderExtensions

EF Core model-configuration extensions that register the audit entity types on a consumer's DbContext.

NotAuditedAttribute

Excludes an entity type or a single property from audit capture.

ServiceCollectionExtensions

Dependency-injection helpers for registering the audit options and user provider.

AuditUser

IAuditValueReader

Supplies stored column values to an IAuditValueSerializer during decode.

Encodes values captured by the audit interceptor into the column(s) it persists on the AuditDetail row, and decodes them back on read.

IAuditValueWriter

Receives the serialized form of a value being captured into an AuditDetail row.

Enums

AuditOperation

Identifies the type of change captured on an AuditHeader row. The underlying byte values are the ASCII codes for 'I', 'U', and 'D'; the EF model converts the enum through char on the way out so the database column literally contains 'I' / 'U' / 'D' when queried directly.
