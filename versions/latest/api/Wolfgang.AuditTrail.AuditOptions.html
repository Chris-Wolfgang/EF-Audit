CaptureDeletedValues

When true, Delete operations write detail rows containing the pre-delete column values. When false (the default), Delete operations write only a header row.

public bool CaptureDeletedValues { get; set; }

Property Value

bool

Remarks

Important — stub-attached deletes: the captured values come from PropertyEntry.OriginalValue, which is meaningful only when the entity was actually loaded (or had its originals seeded) before deletion. The classic "stub delete" pattern — 

context.Set<Customer>().Remove(new Customer { Id = 5 })

— attaches an entity whose non-key properties are CLR defaults, so the detail rows would record those defaults rather than the actual row state. When this flag is on, consumers should load the row first (or call entry.Reload()) before removing it. The captured values are fundamentally a snapshot of what EF Core knew at delete time, not a database read-back.
